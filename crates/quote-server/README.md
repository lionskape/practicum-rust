# quote-server

Сервер синтетических биржевых котировок. Генерирует поток котировок для 110 акций с частотой 10 Гц и транслирует их подписанным клиентам по UDP.

## Запуск

```bash
RUST_LOG=info cargo run -p quote-server
```

С явным указанием адресов:

```bash
RUST_LOG=info cargo run -p quote-server -- \
    --tcp-addr 127.0.0.1:8080 \
    --udp-addr 0.0.0.0:5000
```

## Аргументы командной строки

| Аргумент     | По умолчанию     | Описание                                          |
|--------------|------------------|---------------------------------------------------|
| `--tcp-addr` | `127.0.0.1:8080` | TCP-адрес для приёма клиентских подключений        |
| `--udp-addr` | `0.0.0.0:0`      | UDP-адрес для отправки котировок и приёма PING     |

> Значение `0.0.0.0:0` для UDP означает привязку к любому интерфейсу со случайным свободным портом.

## Переменные окружения

| Переменная | Описание                          | Пример            |
|------------|-----------------------------------|--------------------|
| `RUST_LOG`  | Уровень логирования (`tracing`)  | `info`, `debug`, `quote_server=debug` |

## Протокол

### Рукопожатие (TCP)

1. Клиент подключается к TCP-серверу.
2. Клиент отправляет команду:
   ```
   STREAM udp://HOST:PORT TICKER1,TICKER2,...\n
   ```
3. Сервер отвечает:
   - Успех: `OK SERVER_UDP_ADDR\n`
   - Ошибка: `ERR описание_ошибки\n`
4. TCP-соединение закрывается.

### Поток котировок (UDP)

После успешного рукопожатия сервер отправляет JSON-котировки на UDP-адрес клиента:

```json
{"ticker":"AAPL","price":187.42,"volume":3421,"timestamp":1708617600000}
```

### Keepalive (UDP)

- Клиент должен периодически (каждые 2 с) отправлять `PING` на UDP-сокет сервера.
- Если PING не приходит в течение 5 секунд, сервер отключает клиента.

## Архитектура

```
┌─────────────────────────────────────────────────┐
│                  quote-server                    │
│                                                  │
│  ┌────────────┐    broadcast    ┌─────────────┐  │
│  │  Generator  │──────────────▶│  Client      │──┼──▶ UDP → Client 1
│  │  (10 Гц)   │               │  Registry    │──┼──▶ UDP → Client 2
│  └────────────┘               └─────────────┘  │
│                                                  │
│  ┌────────────┐               ┌─────────────┐  │
│  │ TCP Listener│◀── подписка ──│   Клиенты    │  │
│  └────────────┘               └─────────────┘  │
│                                                  │
│  ┌────────────┐                                  │
│  │ Ping       │◀── UDP PING ── Клиенты          │
│  │ Receiver   │                                  │
│  └────────────┘                                  │
└─────────────────────────────────────────────────┘
```

| Поток           | Назначение                                    |
|-----------------|-----------------------------------------------|
| Generator       | Генерация котировок с моделью случайного блуждания |
| TCP Listener    | Приём подписок клиентов                       |
| Ping Receiver   | Обработка PING-пакетов от клиентов            |
| Client Sender   | По одному на клиента — фильтрация и отправка  |
