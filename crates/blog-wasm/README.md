# blog-wasm

WASM-фронтенд для блог-платформы. Компилируется в WebAssembly и работает в браузере, взаимодействуя с `blog-server` через HTTP API.

## Возможности

- Регистрация и вход через формы в браузере
- Создание постов и просмотр ленты
- Хранение JWT-токена в `localStorage`
- Работа без фреймворков — прямая манипуляция DOM через `web-sys`

## Архитектура

Крейт **не зависит** от `blog-client`, так как `reqwest` и `tonic` несовместимы с WASM. Вместо этого HTTP-запросы выполняются через `gloo-net`, который делегирует в браузерный `fetch()`.

```
src/
├── lib.rs      — точка входа (#[wasm_bindgen(start)])
├── api.rs      — HTTP-запросы к серверу через gloo-net
├── dom.rs      — рендеринг UI и обработка событий
└── storage.rs  — обёртка над localStorage для хранения токена
```

## Требования

- [wasm-pack](https://rustwasm.github.io/wasm-pack/installer/) — для сборки WASM
- Запущенный `blog-server` на `http://localhost:8080`

## Сборка и запуск

```bash
# 1. Установить wasm-pack (если ещё не установлен)
cargo install wasm-pack

# 2. Собрать WASM-модуль
wasm-pack build --target web crates/blog-wasm

# 3. Запустить локальный HTTP-сервер из директории крейта
cd crates/blog-wasm
python3 -m http.server 8000

# 4. Открыть в браузере
open http://localhost:8000
```

## Настройка адреса сервера

По умолчанию WASM-клиент обращается к `http://localhost:8080/api`. Для изменения отредактируйте константу `BASE_URL` в `src/api.rs`:

```rust
const BASE_URL: &str = "http://localhost:8080/api";
```

Сервер должен быть настроен с CORS (уже включён в `blog-server` по умолчанию через `CorsLayer::permissive()`).

## Интерфейс

Приложение имеет два состояния:

**Неавторизованный пользователь** — формы регистрации и входа.

**Авторизованный пользователь** — форма создания поста, кнопка обновления ленты, список постов с пагинацией, кнопка выхода.

## Компиляция в CI

Крейт компилируется на host-target (не для `wasm32-unknown-unknown`) в рамках `cargo ci`. Это работает, потому что `web-sys` предоставляет стабы для non-WASM целей. Полная WASM-сборка выполняется отдельно через `wasm-pack`.
